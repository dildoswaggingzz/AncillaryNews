Historical backfill

The ingestor's scheduled poller only ever fetches the most recent handful of records per dataset, so market_data_history only has real depth back to whenever it was first brought up. Use this to backfill weeks of real history for the datasets the BESS backtest simulator (/dashboard/bess/new) reads, via Energinet's public start/end date-range API. Free -- pure Energinet HTTP polling + Postgres writes, no Anthropic API calls. Safe to re-run (see shared/backfill.py).

Chunk size matters for high-volume datasets. Energinet's API truncates (rather than erroring) when a date-range chunk has more records than the per-chunk limit -- a truncated chunk still "succeeds" with a plausible row count, so it will not look like a failure. afrr_energy_activation is the known case: it runs ~172,400 records/day, so the default 7-day chunk (~1.2M records) is truncated to roughly a quarter of the window's real data. It is in the default (blank Datasets field) BESS dataset set, so leaving Datasets blank backfills it at the truncating default unless you lower Chunk days. Use chunk_days=1 for afrr_energy_activation (and afrr_border_atc if backfilling the forecasting datasets). If a run reports any truncated chunks below, re-run just that dataset with a smaller chunk size.
